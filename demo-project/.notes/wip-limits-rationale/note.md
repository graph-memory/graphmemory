# WIP Limits Rationale

WIP (Work In Progress) limits on kanban columns prevent team overload. When a column reaches its limit, no new tasks can be moved in. This encourages finishing work before starting new items. The default limits are: in_progress=5, review=3. Limits are configurable per project.
