# Internationalization (i18n)

The ShopFlow Web Store supports multiple languages, locales, and text directions. This document covers the translation infrastructure, locale management, RTL support, and formatting conventions.

## Architecture Overview

```
┌──────────────────────────────────┐
│         React Components         │
│   useTranslation() / <Trans />   │
├──────────────────────────────────┤
│         i18n Provider            │
│   (locale state + formatters)    │
├────────┬────────┬────────────────┤
│  en-US │  es-ES │  ar-SA  │ ... │
│  .json │  .json │  .json  │     │
└────────┴────────┴────────────────┘
```

## Translation Files

Translation strings are stored as flat JSON files in `src/locales/{locale}.json`:

```json
{
  "header.shop": "Shop",
  "header.categories": "Categories",
  "header.cart": "Cart",
  "product.addToCart": "Add to Cart",
  "product.outOfStock": "Out of Stock",
  "product.lowStock": "Only {count} left",
  "cart.empty": "Your cart is empty. Start shopping!",
  "cart.subtotal": "Subtotal",
  "cart.checkout": "Proceed to Checkout",
  "checkout.step.address": "Shipping Address",
  "checkout.step.shipping": "Shipping Method",
  "checkout.step.payment": "Payment",
  "checkout.step.confirmation": "Review Your Order",
  "search.placeholder": "Search products...",
  "search.noResults": "No results found for \"{query}\""
}
```

### Key Naming Convention
- Dot-separated namespace: `{component}.{element}.{variant}`
- Use descriptive English keys, not numeric IDs
- Pluralization uses ICU MessageFormat: `{count, plural, one {# item} other {# items}}`

## Translation Workflow

### Adding New Strings
1. Add the English string to `en-US.json` with a descriptive key
2. Use `useTranslation()` in the component to reference the key
3. Run `npm run i18n:extract` to detect untranslated keys in other locales
4. Translators fill in missing keys via the Crowdin integration
5. Run `npm run i18n:pull` to download updated translations
6. CI validates that all locales have complete coverage

### Using Translations in Components

```typescript
import { useTranslation } from '@/hooks/useTranslation';

function CartEmpty() {
  const { t } = useTranslation();
  return <p>{t('cart.empty')}</p>;
}
```

### Interpolation

```typescript
// Simple variable
t('product.lowStock', { count: 3 })
// → "Only 3 left"

// Pluralization
t('cart.itemCount', { count: items.length })
// → "1 item" or "5 items"
```

## RTL Support

### CSS Logical Properties
All layout styles use CSS logical properties instead of physical directions:

```css
/* Correct */
.card { margin-inline-start: 16px; padding-block: 8px; }

/* Avoid */
.card { margin-left: 16px; padding-top: 8px; }
```

### Direction Attribute
The `<html>` element receives `dir="rtl"` or `dir="ltr"` based on the active locale. The i18n provider sets this automatically on locale change.

### Bidirectional Text
User-generated content (product descriptions, reviews) is wrapped in `<bdi>` elements to isolate text direction. Numbers and brand names embedded in RTL text use `<span dir="ltr">` to maintain correct display.

### Icon Mirroring
Directional icons (arrows, chevrons, navigation) are mirrored in RTL mode using `transform: scaleX(-1)`. Non-directional icons (cart, search, star) are not mirrored.

## Number and Date Formatting

### Prices
All prices are formatted using `Intl.NumberFormat` with the user's locale and the product's currency:

```typescript
function formatPrice(price: number, currency: string, locale: string): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(price);
}
// en-US: "$29.99"
// de-DE: "29,99 $"
// ja-JP: "$29.99"
```

### Dates
Order dates and timestamps use `Intl.DateTimeFormat`:

```typescript
function formatDate(date: string, locale: string): string {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(new Date(date));
}
// en-US: "March 15, 2026"
// de-DE: "15. März 2026"
// ja-JP: "2026年3月15日"
```

### Relative Time
"Added 2 hours ago" style timestamps use `Intl.RelativeTimeFormat` for locale-aware relative time display in cart and order history views.

## Locale Detection

1. Check URL parameter: `?locale=fr-FR`
2. Check stored preference in localStorage
3. Check browser `navigator.language`
4. Fall back to `en-US`

The detected locale is stored in user preferences via the `storage.ts` service and persists across sessions.

## Testing

- **Pseudo-localization**: A `pseudo` locale expands all strings by 30% and wraps in brackets `[Ëxpâñdëd Strîñg]` to detect hardcoded strings and layout overflow
- **RTL visual testing**: Storybook stories render each component in both LTR and RTL modes
- **Missing key detection**: CI fails if any locale has untranslated keys
- **Snapshot testing**: Translation file structure is snapshot-tested to catch accidental key removals
