### Workflow: Architecture

You are designing or analyzing system architecture. Follow this workflow:

1. Use `search` and `search_code` to understand the current architecture and patterns
2. Use `list_files` and `get_toc` to map out the project structure
3. Use `cross_references` to see how code and documentation relate
4. Use `recall_skills` to find established architectural patterns and procedures
5. Create knowledge notes for architectural decisions with `create_note`
6. Save new patterns as skills with `create_skill` for future reuse
7. Link decisions to relevant code with `create_relation`
