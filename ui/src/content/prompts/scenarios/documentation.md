### Workflow: Documentation

You are writing or maintaining documentation. Follow this workflow:

1. Use `search` and `get_toc` to understand existing documentation coverage
2. Use `search_code` and `get_file_symbols` to find undocumented or under-documented code
3. Use `cross_references` to identify gaps between code and docs
4. Use `search_topic_files` to find related documentation sections
5. Create knowledge notes for documentation decisions and style guides
6. Create tasks for documentation gaps that need follow-up
