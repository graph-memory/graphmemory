# Analytics

The analytics module provides business intelligence for the ShopFlow platform. It aggregates order, product, and customer data into actionable visualizations and metrics.

## Revenue Tracking

### Revenue Trend Chart

The primary revenue visualization is a bar chart showing revenue over time. The chart supports three granularity levels based on the selected date range:

| Date Range | Granularity | Example |
|------------|-------------|---------|
| 1-14 days | Daily | Revenue per day |
| 15-90 days | Weekly | Revenue per week |
| 91+ days | Monthly | Revenue per month |

Revenue is calculated from completed orders only (status: `delivered`). Cancelled and refunded orders are excluded from revenue totals but tracked separately for reporting.

### Revenue Metrics

- **Total Revenue** — Sum of all completed order amounts
- **Revenue Growth** — Percentage change comparing the first half vs second half of the selected period
- **Average Order Value (AOV)** — Total revenue divided by total completed orders
- **Revenue per Customer** — Total revenue divided by unique customers

## Order Analytics

### Order Volume

A bar chart showing the number of orders placed per time period, using the same granularity rules as revenue. This helps identify demand patterns and seasonal trends.

### Order Status Distribution

Breakdown of orders by current status, useful for identifying bottlenecks:

- Pending orders that need attention
- Processing orders in the fulfillment pipeline
- Shipped orders in transit
- Cancellation and refund rates

## Top Products

A ranked list of the best-performing products in the selected period, measured by:

1. **Units Sold** — Total quantity of the product sold across all orders
2. **Revenue** — Total revenue generated by the product
3. **Conversion Rate** — Percentage of product page views that resulted in a purchase

The top products list shows the top 10 by default, with an option to expand to the full ranking.

## Conversion Funnel

The conversion funnel tracks user progression through the purchase flow:

```
Visit → Product View → Add to Cart → Checkout Started → Purchase Completed
```

### Funnel Stages

| Stage | Description | Tracking |
|-------|-------------|----------|
| Visit | Any session on the storefront | Page view event |
| Product View | Viewed a product detail page | Page view with product ID |
| Add to Cart | Added at least one item to cart | Cart mutation event |
| Checkout Started | Began the checkout flow | Checkout init event |
| Purchase Completed | Successfully placed an order | Order created event |

### Funnel Metrics

Each stage shows:
- **Count** — Number of unique users who reached that stage
- **Conversion Rate** — Percentage of users from the previous stage who progressed
- **Drop-off Rate** — Percentage of users who left at this stage

The funnel visualization uses a tapered bar chart where each stage is narrower than the previous one, proportional to the conversion rate.

## Cohort Analysis

Cohort analysis groups customers by their first purchase month and tracks their repeat purchase behavior over subsequent months.

### Retention Table

A matrix where:
- Rows represent cohorts (month of first purchase)
- Columns represent months since first purchase (Month 0, Month 1, Month 2, ...)
- Cells show the percentage of the cohort that made a purchase in that month

This helps measure customer loyalty and the effectiveness of retention campaigns.

## Custom Date Ranges

### Presets

Four quick-select presets are available:

| Preset | Range |
|--------|-------|
| 7d | Last 7 days |
| 30d | Last 30 days |
| 90d | Last 90 days |
| 12m | Last 12 months |

### Custom Range

Users can also select a custom start and end date using date pickers. The maximum selectable range is 2 years to keep query performance reasonable.

### Period Comparison

All metrics show a comparison against the equivalent previous period. For example, selecting "Last 30 days" compares against the 30 days before that. Growth percentages are calculated as:

```
growth = ((current - previous) / previous) * 100
```

## Data Pipeline

Analytics data is aggregated from the following sources:

1. **Order events** — Real-time from the order processing service
2. **Page view events** — Batched from the storefront analytics tracker
3. **Cart events** — Real-time from the cart service

Data is pre-aggregated into daily rollup tables for fast querying. The analytics API endpoint computes further aggregations (weekly, monthly) on the fly from the daily rollups.

### Freshness

- Revenue and order metrics update in near real-time (within 1 minute)
- Funnel data has a 5-minute delay due to event batching
- Cohort analysis is computed nightly and cached

## Growth Rate Calculation

The `computeGrowthRate` function splits the selected time series at the midpoint and compares the sum of the second half against the first half:

```typescript
const midpoint = Math.floor(series.length / 2);
const firstHalf = series.slice(0, midpoint).reduce((sum, p) => sum + p.value, 0);
const secondHalf = series.slice(midpoint).reduce((sum, p) => sum + p.value, 0);
return ((secondHalf - firstHalf) / firstHalf) * 100;
```

This provides a simple but effective trend indicator without requiring a full previous-period comparison dataset.
