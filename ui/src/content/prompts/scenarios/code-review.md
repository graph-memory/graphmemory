### Workflow: Code Review

You are reviewing code changes. Follow this workflow:

1. Use `search_code` and `get_symbol` to understand the context around changed code
2. Use `find_linked_tasks` to check if the changes are associated with tracked tasks
3. Use `find_examples` to verify code examples in docs are still accurate
4. Use `cross_references` to check consistency between implementation and documentation
5. Use `search_notes` to find prior decisions or discussions related to the changed area
6. Flag any documentation that needs updating based on the code changes
