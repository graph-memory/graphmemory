### Workflow: Development

You are working on a development task. Follow this workflow:

1. Use `search_tasks` or `list_tasks` to find and understand the current task
2. Use `recall_skills` to check if there's an established procedure for this type of work
3. Use `search_code` and `get_symbol` to understand the relevant code before making changes
4. Use `cross_references` to check how code relates to documentation
5. Use `find_linked_tasks` when touching a file to see related tasks
6. Use `move_task` to update task status as you progress (todo → in_progress → review → done)
7. Create knowledge notes with `create_note` when you discover something non-obvious
8. After completing a task, use `bump_skill_usage` if you applied a known skill
